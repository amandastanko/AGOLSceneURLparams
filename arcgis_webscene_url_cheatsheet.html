URL Parameters &
Web Scenes

A cheat sheet for steering Scene Viewer from the address bar — viewpoints, slides, layers, embeds.
3D / GIS Compiled from
doc.arcgis.com
& esri.com/blog
Updated 2026
URL parameters are key/value pairs that follow a question mark in a Scene Viewer link, separated by ampersands. They let you preconfigure a scene the moment it loads — jumping straight to a slide, framing a specific camera angle, adding layers on the fly, or stripping the UI for a clean embed. The base URL is always https://www.arcgis.com/home/webscene/viewer.html (or your portal equivalent).

§ 01

Anatomy of a Scene URL

Syntax 
// The basic shape
https://www.arcgis.com/home/webscene/viewer.html?webscene=<itemID>&ui=min#<slide-number>
                                                 │           │             │
                                                 │           │             └── fragment (slide index)
                                                 │           └────────────────  additional parameters
                                                 └────────────────────────────  first parameter

Three connectors do all the work: ? opens the query string, & joins additional parameters, and # introduces the slide fragment (which behaves slightly differently — see §04).

§ 02

The Core Parameters

Reference
?webscene=<itemID>
Scene Loader

Opens a saved Web Scene by its ArcGIS Online or Portal item ID. The ID is the 32-character hex string from the item's page.

?webscene=12f2c243d6a346beaee9702d96c19e7c
?url=<service-url>
Direct Service

Loads a service directly into a blank scene — useful when you have a SceneServer, FeatureServer, or MapServer URL but no saved item.

?url=https://tiles.arcgis.com/tiles/.../Buildings_Portland/SceneServer
?layers=<id1>,<id2>
Layer Stacker

Adds one or more layers to the scene by item ID. Combine multiple IDs with commas — no spaces. Works alongside webscene= to overlay extra content on an existing scene.

?layers=9e2f2b544c954fda9cd13b7f3e6eebce,adfe292a67f8471a9d8230ef93294414
&ui=min
Embed Mode

Strips Scene Viewer down to a minimal UI — ideal for iframe embeds in third-party pages where you don't want the full toolbar competing with your design.

&ui=min
§ 03

The Viewpoint Parameter

Camera Control

A viewpoint pins the camera to a specific location, altitude, and angle. The format packs five numbers into a single string with two delimiters — a comma list, then a semicolon, then another comma list.

&viewpoint=cam:longitude,latitude,altitude;heading,tilt
▸ Decoded
cam: prefix indicating a camera viewpoint (rather than an extent)
longitude camera position east–west, in decimal degrees
latitude camera position north–south, in decimal degrees
altitude camera height in meters above the ellipsoid
heading compass direction the camera faces, 0–360° (0 = north)
tilt camera pitch, 0° = looking straight down, 90° = horizontal 
// Example: Inner Alster Lake, Hamburg
&viewpoint=cam:9.98717314,53.54101423,847.391;23.206,65.41

Don't hand-craft these numbers. There's a better way ↓

Globie's tip

Frame your scene visually, click Share, uncheck Short link, and copy the long-form URL — the viewpoint=cam: string is already in there, ready to paste into any other URL you're building.

§ 04

Jumping to a Slide

Hash Fragment
#<slide-number>
Fragment

Opens the scene to the Nth slide (1-indexed). Note this is a hash fragment, not a query parameter — it sits after any &-joined parameters, with no ampersand of its own.

// Open scene to the 3rd slide
https://www.arcgis.com/home/webscene/viewer.html?webscene=6c7a0d4f03d2444fb1f4fa9e85a96044#3
A note on slide transition speed

Esri's documented URL parameters for Scene Viewer do not include controls for slide transition speed, autoplay timing, or animation easing. The #<n> fragment jumps to a slide using Scene Viewer's default transition, and that's the extent of slide control available from the URL.

If you need to choreograph transitions — custom durations, eased camera moves, autoplay loops — that lives in the ArcGIS Maps SDK for JavaScript, using view.goTo(target, { duration, easing }) against the scene's presentation.slides collection. Build a small custom viewer if URL-driven timing is the goal.

§ 05

Quick Reference

Lookup Table
Parameter Position Effect
?webscene=<id> First / query Loads a saved scene by ArcGIS item ID.
?url=<service> First / query Loads a service URL directly into a fresh scene.
&layers=<id>,<id> Query Adds one or more layers by item ID; comma-separated.
&viewpoint=cam:… Query Sets the initial camera position, heading, and tilt.
&ui=min Query Minimizes the Scene Viewer chrome for clean embeds.
#<n> End / fragment Opens directly to the Nth slide (1-indexed).

§ 07

Field Notes

Gotchas & Tips
  1. Order matters only at the boundaries. The first parameter takes a ?; all others use &. The slide fragment #n always goes last.
  2. URL-encode when embedding. Browsers encode address-bar URLs automatically, but iframes and email-embedded links often need manual encoding (e.g., URLEncoder.io).
  3. The viewpoint format is fragile. Commas between coordinates, semicolon before heading/tilt. Swap them and the scene silently falls back to its default extent.
  4. Portal URLs differ. ArcGIS Enterprise users replace www.arcgis.com with their portal host, e.g. myportal.domain.com/portal.
  5. Capture viewpoints, don't compute them. Use Share → uncheck Short link → copy. The long-form URL hands you a ready-made viewpoint=cam: string.
  6. The layers parameter expects item IDs. For a raw service URL, use url= instead — different parameter, different shape.
  7. Slide numbers are 1-indexed. The first slide is #1, not #0.
  8. Test in an incognito window. Cached sessions can mask parameter failures when you're iterating on a shared link.
